# 统计区域与列表间隙优化

## 问题描述
用户反馈待审核统计和下方的待审核列表中间的间隙太大，需要减少这两个区域之间的距离。

## 优化调整

### 1. 统计容器下边距调整
**文件位置**: `pages/site/facility.vue` - `.stats-container`

**调整内容**:
- 外边距：`20rpx 16rpx` → `20rpx 16rpx 8rpx 16rpx`
- 下边距从 20rpx 减少到 8rpx，减少了 12rpx

### 2. 待审核列表标题上边距调整
**文件位置**: `pages/site/facility.vue` - `.audit-list-title`

**调整内容**:
- 上边距：`0` → `-8rpx`
- 使用负边距让标题向上移动 8rpx

### 3. 审核列表容器上边距调整
**文件位置**: `pages/site/facility.vue` - `.audit-list`

**调整内容**:
- 上边距：`-24rpx` → `-32rpx`
- 进一步向上移动 8rpx

## 具体代码变更

### CSS样式调整
```scss
/* 统计数据展示区域样式 */
.stats-container {
    margin: 20rpx 16rpx 8rpx 16rpx;  // 原: 20rpx 16rpx
}

/* 待审核列表标题特殊样式 */
.audit-list-title {
    margin-top: -8rpx !important;   // 原: 0
}

/* 卡片列表紧凑化 */
.audit-list {
    margin-top: -32rpx;              // 原: -24rpx
}
```

## 优化效果

### 间隙减少量
- 统计容器下边距：减少 12rpx
- 列表标题上移：减少 8rpx
- 列表容器上移：减少 8rpx

**总计减少约 28rpx 的垂直间隙**

### 视觉效果改进

**之前**:
- ❌ 统计区域和列表之间间隙过大
- ❌ 视觉上两个区域分离感强
- ❌ 空间利用率不高

**之后**:
- ✅ 统计区域和列表紧密连接
- ✅ 视觉上形成统一的功能区域
- ✅ 空间利用更加高效

### 布局层次
```
待审批统计区域
    ↓ (减少28rpx间隙)
待审核列表标题
    ↓ (紧密连接)
搜索框区域
    ↓ (紧密连接)
数据列表区域
```

## 用户体验提升

1. **视觉连贯性**: 统计和列表形成更紧密的功能组合
2. **空间效率**: 减少不必要的空白，提升内容密度
3. **操作流畅性**: 用户视线从统计到列表的转移更自然
4. **信息关联性**: 统计数据和具体列表的关联性更强

这些调整让统计区域和待审核列表之间的关系更加紧密，提升了整体的视觉连贯性和空间利用效率。
